# Academy-Top-Manual-testing

#### Решения домашних заданий "Мануальное тестирование"

## Homework 01

### Тема: "Введение в тестирование. Ошибка, дефект, сбой. Роль QA"

**Задание 1. Разбор трех ситуаций**    
Выберите игру, приложение или сайт.    
Найдите 3 проблемные ситуации и для каждой заполните:
1. Контекст (платформа, версия, условия).
2. Предполагаемый error (где могла возникнуть человеческая ошибка).
3. Defect (что именно в системе работает неправильно).
4. Failure (что видит пользователь).
5. Риск для пользователя.    
**Минимальный объем: 3 полноценных ситуации.**

**Задание 2. Один подробный мини-репорт**    
Выберите 1 ситуацию из задания 1 и оформите подробнее:
1. Название проблемы.
2. Окружение.
3. Шаги воспроизведения (не менее 4 шагов).
4. Ожидаемый результат.
5. Фактический результат.
6. Повторяемость (например: 5/5, 2/5).
7. Какой это failure.
8. Какой defect за ним предполагается.

**Задание 3. Роль QA в кейсе**    
К этому же примеру напишите коротко (5-8 предложений):
1. Что сделал QA правильно.
2. Какие данные еще стоит добавить.
3. Как такой репорт поможет команде быстрее исправить проблему.

**Дополнительная часть (по желанию):**    
Сравните два ваших кейса и ответьте:
1. В каком кейсе риск выше и почему?
2. Где проще воспроизвести failure?
3. Какой кейс вы бы проверяли первым на регрессии?

## Homework 02

### Тема: "Качество ПО: ожидание и реальность"

**Задание 1. Три кейса качества**    
Выберите одну игру, приложение или сайт.    
Найдите 3 предполагаемых дефекта и для каждого заполните:
1. Контекст (платформа, версия, условия).
2. Ожидание.
3. Факт.
4. Риск для пользователя.
5. Severity.
6. Priority.    
Минимальный объем: 3 полноценных кейса.

**Задание 2. Подробный баг-репорт**    
Выберите 1 кейс из задания 1 и оформите его как полноценный баг-репорт:
1. Название дефекта.
2. Окружение.
3. Шаги воспроизведения (не менее 4 шагов).
4. Ожидаемый результат.
5. Фактический результат.
6. Повторяемость (например: 5/5, 3/5).
7. Риск.
8. Severity/Priority.

**Задание 3. Временное решение**    
Для этого же дефекта предложите 1-2 временные меры до исправления в коде.    
Примеры::
- предупреждение в интерфейсе;
- обходной пользовательский сценарий;
- ограничение проблемного действия;
- рекомендация по сохранению прогресса.

**Дополнительная часть (по желанию):**    
Сравните два дефекта из ваших кейсов и ответьте:
1. Почему один дефект приоритетнее другого?
2. Какой дефект сильнее влияет на удержание пользователя?
3. Что вы бы исправили в первую очередь и почему?
